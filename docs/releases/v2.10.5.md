# Release v2.10.5

**Date:** 2026-04-16
**Type:** Patch — Qwen3 model upgrade to Qwen3.6
**Previous:** v2.10.4

## Highlights

- Qwen3 premium default upgraded from `qwen3.5-plus` to `qwen3.6-plus` (80% cheaper)
- Qwen3 standard tier now uses the open-weight `qwen3.6-35b-a3b` (MoE, 35B total / 3B active, Apache 2.0)
- Eliminated stale hardcoded model mappings in `get_economic_model()`

## What's New

### Qwen3.6 Model Upgrade

Alibaba released Qwen3.6-35B-A3B on April 16, 2026 — a sparse Mixture-of-Experts model with 35B total parameters but only 3B active during inference. The model scores 51.5 on Terminal-Bench 2.0 and 73.4 on SWE-bench Verified.

| Tier | Before (v2.10.4) | After (v2.10.5) |
|------|-------------------|------------------|
| Premium | qwen3.5-plus ($1.60/$6.40 per M) | qwen3.6-plus ($0.325/$1.95 per M) |
| Standard | qwen3.5-plus ($1.60/$6.40 per M) | qwen3.6-35b-a3b ($0.163/$0.90 per M) |
| Economy | qwen3-32b | qwen3-32b (unchanged) |

The premium tier (`qwen3.6-plus`) is Alibaba's hosted API with 1M context window. The standard tier (`qwen3.6-35b-a3b`) is the open-weight variant, also available on Ollama as `qwen3.6:35b-a3b` (24GB).

### DRY Refactor: get_economic_model()

`get_economic_model()` in `lib/costs.sh` had a hardcoded case statement with stale model names (e.g., `qwen-turbo`, `claude-3-haiku`, `gemini-2.5-flash`). Refactored to delegate to `get_model_for_tier()` in `config.sh`, matching the pattern already used by `get_premium_model()` in `lib/routing.sh`.

### Version Fix

Fixed `AI_CONSULTANTS_VERSION` in `config.sh` which was stuck at `2.10.0` since the npm packaging release.

## Upgrade Guide

curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash

No configuration changes required. Existing `QWEN3_MODEL` overrides continue to work.
