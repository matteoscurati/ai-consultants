# Release v2.24.0

**Date:** 2026-07-20
**Type:** Minor — installing now removes slash commands that were deleted in earlier releases
**Previous:** v2.23.0

## Highlights

- Installing or updating now **removes** `ai-consultants:*.md` slash commands this project no longer ships. Previously they were only ever copied in, so a command deleted upstream stayed on your machine forever.
- Ten such commands have been dead since v2.10.0 and are still installed on every setup predating it. One of them, `ai-consultants:config-features`, tells you to set `ENABLE_REFLECTION` — removed in v2.23.0, and now rejected by `configure --set`.
- **Read the breaking-change note below if you wrote your own commands under the `ai-consultants:` prefix.**
- Both entry points are fixed: `curl | bash` (`scripts/install.sh`) and npx (`ai-consultants install`).
- `scripts/release.sh` is no longer shipped in the npm package.

## What's New

### Stale slash commands are pruned on install and update

Both installers used to `cp` the current commands into `~/.claude/commands` and nothing else. Nothing ever removed one that had been deleted from the project, so obsolete commands accumulated and stayed invocable.

Measured on a real installation: **14 `ai-consultants:*.md` files present, 4 of which this project still ships.** The other ten were removed by the v2.10.0 command consolidation, thirteen releases ago:

```
ask-experts       config-check      config-preset     config-strategy
config-api        config-features   config-status     config-wizard
config-budget     config-personas
```

They are not merely clutter. `config-features` instructs you to set `ENABLE_REFLECTION`, a setting v2.23.0 removed; `ai-consultants configure --set ENABLE_REFLECTION=...` now exits non-zero. Following that command's instructions fails.

Both paths now prune before copying and report what they removed:

```
Pruned 10 slash command(s) removed in a previous release
Installed 4 slash commands to /Users/you/.claude/commands
```

### `scripts/release.sh` removed from the npm package

Maintainer-only version-bump automation. `bin/ai-consultants` does not route it, so it was never reachable from the CLI — `npx ai-consultants release` starts a consultation, it does not run the script.

Other scripts that look maintainer-only are **deliberately kept**, having checked what depends on them: `roster_audit.sh` backs the shipped `roster-audit` slash command and `doctor --roster-audit`; `roster_calibrate.sh`, `taste_elo.sh` and `run_calibration.sh` are documented for your own use in `docs/SMART_ROUTING.md`; `install-hooks.sh` is wired to a `package.json` script.

## Breaking Changes

**The prune removes any `ai-consultants:*.md` this project does not ship — including a command you wrote yourself under that prefix.**

There is no way to distinguish a command you authored in this namespace from one of ours that was deleted upstream: both are simply files we do not ship. The `ai-consultants:` prefix is this project's namespace, and the prune treats it as such.

If you have custom commands there, **rename them out of the prefix before upgrading**:

```bash
mv ~/.claude/commands/ai-consultants:mine.md ~/.claude/commands/my-consultants:mine.md
```

Commands belonging to other tools are never considered — only the `ai-consultants:` prefix is examined.

## Upgrade Guide

```bash
# npx (no install)
npx ai-consultants "question"

# refresh the slash commands (this is what prunes)
npx ai-consultants install

# git clone install
cd ~/.claude/skills/ai-consultants && git pull

# curl | bash install
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

No configuration changes required.

Note for `git pull` users: pulling updates the checkout but does **not** re-run the installer, so it does not prune. Run `scripts/install.sh` or `ai-consultants install` afterwards to refresh the installed commands.
