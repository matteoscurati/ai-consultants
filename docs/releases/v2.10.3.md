# Release v2.10.3

**Date:** 2026-03-21
**Type:** Patch — Model updates, API migration, cleanup
**Previous:** v2.10.2

## Highlights

- Grok premium model upgraded to `grok-4.20-0309-reasoning`
- GLM API migrated from `open.bigmodel.cn` to `api.z.ai/api/coding/paas/v4`
- Removed non-functional MiniMax highspeed models
- Fixed stale GLM URL fallbacks in `common.sh` and `configure.sh`

## What's New

### Grok Model Upgrade

Grok premium model updated from `grok-4-1-fast-reasoning` to `grok-4.20-0309-reasoning`. Standard (`grok-3`) and economy (`grok-3-mini`) tiers remain unchanged.

| Tier | Before (v2.10.2) | After (v2.10.3) |
|------|-------------------|------------------|
| Premium | grok-4-1-fast-reasoning | grok-4.20-0309-reasoning |
| Standard | grok-3 | grok-3 |
| Economy | grok-3-mini | grok-3-mini |

### GLM API Migration

Zhipu AI migrated their API from `open.bigmodel.cn` to `api.z.ai`. The default endpoint is now `https://api.z.ai/api/coding/paas/v4/chat/completions`. Updated in:

- `config.sh` (default)
- `common.sh` (fallback in `get_api_url()`)
- `configure.sh` (wizard URL array)
- `.env.example` (example config)

### Removed Non-Functional Models

Removed models not supported by the current MiniMax token plan:

- `MiniMax-M2.7-highspeed`
- `MiniMax-M2.5-highspeed`
- `grok-beta` (legacy, no longer recognized by xAI API)

### Bug Fixes

- Fixed stale GLM URL fallback in `get_api_url()` (`common.sh`) — would have routed to old endpoint if `GLM_API_URL` env var was unset
- Fixed stale GLM URL in configure wizard (`configure.sh`) — would have written old endpoint to `.env`
- Fixed duplicate `minimax-m2.5` entry in `cost_rates.json` with conflicting input rates ($0.30 vs $0.50)

## Breaking Changes

None.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull

# If installed via curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

No configuration changes required. Existing `GROK_MODEL` and `GLM_API_URL` overrides continue to work.
