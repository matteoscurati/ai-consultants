# Release v2.14.1

**Date:** 2026-05-13
**Type:** Patch — Developer experience
**Previous:** v2.14.0

## Highlights

- **Pre-commit hook for shellcheck** prevents the class of CI failure that hit v2.14.0 (SC2164 in a new script that passed `bash -n` but not shellcheck). Install once per checkout with `npm run install-hooks`.
- **`npm run lint`** runs the full-repo shellcheck invocation that CI uses, for ad-hoc verification before pushing.
- No behavioral changes for end users.

## What's New

### `scripts/hooks/pre-commit`

A git pre-commit hook that runs `shellcheck` on staged `.sh` files under `scripts/` (matching the CI glob `scripts/*.sh scripts/lib/*.sh`). The invocation mirrors `.github/workflows/ci.yml` exactly:

```bash
shellcheck -S warning -x -e SC1091,SC1090,SC2034,SC2155 <staged files>
```

So any warning that would fail the CI lint job will also block the local commit, and conversely any warning the project has chosen to exclude is also excluded locally.

Behavior:

- **No staged `.sh` files** → exits clean immediately (zero overhead on doc-only commits).
- **`shellcheck` not on PATH** → prints a one-line warning and lets the commit through. Local convenience only; CI still catches the issues.
- **Failure** → prints the offending lines, the bypass hint, and exits 1.
- **Bypass** → `git commit --no-verify` (intentional escape hatch).

### `scripts/install-hooks.sh`

Idempotent installer. Copies `scripts/hooks/pre-commit` into `.git/hooks/pre-commit` and `chmod +x` it. If an existing different hook is present, backs it up to `.git/hooks/pre-commit.backup.<timestamp>` rather than clobbering — set `FORCE=1` to skip the backup. Silent no-op outside a git checkout (safe for npm tarball consumption — `prepare` and similar hooks won't error).

### `package.json` scripts

```json
{
  "scripts": {
    "test": "bash scripts/test_all.sh",
    "install-hooks": "bash scripts/install-hooks.sh",
    "lint": "shellcheck -S warning -x -e SC1091,SC1090,SC2034,SC2155 scripts/*.sh scripts/lib/*.sh"
  }
}
```

`npm run lint` is the manual equivalent of what CI runs. Useful when you want to lint the whole tree without staging.

## Breaking Changes

None.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull
npm run install-hooks   # one-time per checkout

# If installed via curl | bash or npx — nothing to do (the hook only applies to contributors)
```

## Why this is a patch, not a minor

The hook is contributor-only tooling. It doesn't change runtime behavior, doesn't add features visible to consumers, and doesn't touch any consultant or prompt code. Semver patch is the right level.
