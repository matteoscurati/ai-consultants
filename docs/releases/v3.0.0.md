# Release v3.0.0

**Date:** 2026-07-23
**Type:** Major — removes the deliberation/consensus machinery; default output is now the coverage union
**Previous:** v2.25.2

## Highlights

- **The panel premise was tested and the answer is task-dependent.** A held-out coverage experiment found a single strong model saturates *convergent* defect-finding (it caught 19/19 hard bugs, including real CVEs and a concurrency bug found by formal verification), while a diverse cross-vendor panel materially out-covers it on *breadth* questions — "what could go wrong?", enumerate the risks — winning as a **union of distinct answers**, not through deliberation.
- **Default output is now the `coverage` union** — the deduplicated set of every distinct point, recommendation, risk, and edge case across the panel, preserving what only one model raised. No more voted single recommendation by default.
- **The deliberation/consensus machinery is gone**: multi-round debate, dynamic orchestration, voting/consensus scoring, capability weighting, panic mode, peer review, stance consensus, and the capability-calibration tooling.
- **What made the panel good is kept**: diverse parallel fan-out to all 11 consultants, category-affinity routing, doctor, cost tracking, quorum/health gate, and every adapter.
- **Simpler and smaller**: 19 files removed (~-4,600 lines), `npm test` 22 → 15 suites, all green.

## What's New

### Coverage synthesis (the new default)

The pipeline is now **classify → route → parallel fan-out → coverage synthesis**. Every selected consultant answers once, in parallel; the synthesizer returns the *union* of their distinct points rather than picking a winner or averaging a vote.

| Strategy | Behavior |
|---|---|
| `coverage` (default) | Union of every distinct point/recommendation/risk across the panel; credits what only one model raised |
| `compare_only` | Each consultant side-by-side, no synthesized union |
| `majority` | A single blended recommendation, weighting all equally |
| `risk_averse` / `security_first` / `cost_capped` | Emphasis variants |

Override per run with `--strategy`, or persist with `DEFAULT_STRATEGY`.

### Why

Two independent reviews had judged the panel's premise weak: voting and consensus measure *agreement*, not correctness, and the repo had no held-out comparison against one strong model. The A/W/C coverage experiment (maintainer instrumentation, not shipped) measured, on deep 60-point rubrics across three domains:

| task | single model | self-consistency (same model ×k) | diverse panel (union) |
|---|---|---|---|
| convergent defect-finding | 19/19 (saturates) | — | no headroom to add |
| breadth / enumeration | 51% | 70% | **93%** |

The panel wins on breadth, and it wins as the raw union with deliberation **off** — so the diverse fan-out is the value and the averaging machinery is not.

## Breaking Changes

- **Default output shape changed** from a voted recommendation to the coverage union.
- **Removed features and their config** (setting any of these via `configure --set` now fails closed): `ENABLE_DEBATE`/`DEBATE_ROUNDS`, `ORCHESTRATION_MODE`/`CONVERGENCE_*`, `ENABLE_DEBATE_OPTIMIZATION`, `ENABLE_PANIC_MODE`/`PANIC_*`, `ENABLE_ADVERSARIAL_VERIFY`, `ENABLE_PEER_REVIEW`/`PEER_REVIEW_MIN_RESPONSES`, `ENABLE_CAPABILITY_WEIGHTING`/`ENABLE_CAPABILITY_ROUTING`/`CAPABILITY_*`, `ENABLE_STANCE_CONSENSUS`/`STANCE_*`.
- **Removed outputs**: `voting.json`, `round_N/`, `panic_diagnosis.json`.
- **Removed slash commands**: `/ai-consultants:debate`, `/ai-consultants:roster-audit` (installers prune them on update).
- **Removed tooling**: `roster_audit.sh`, `roster_calibrate.sh`, `run_calibration.sh`, `taste_elo.sh`, `doctor --roster-audit`.
- The affinity matrix keeps its category→consultant routing scores; the `capabilities`/`category_axis` axes are removed.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull

# If installed via curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash

# If installed via npx (no install step)
npx ai-consultants@latest "your question"
```

No configuration change is required. If your `.env` sets a removed variable, delete it — the tool now fans out and returns the coverage union by default. To get a single recommendation instead, use `--strategy majority`; for a raw side-by-side, `--strategy compare_only`.

## Note on the evidence

The breadth result is **directional** (n=3, hand-validated) alongside the n=19 defect-finding result. The cut is a considered bet on that evidence; a statistically binding breadth claim would need n≥15–20 deep-rubric items (the experiment harness exists on the `experiment/coverage-metric` branch).
