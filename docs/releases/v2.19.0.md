# Release v2.19.0

**Date:** 2026-06-26
**Type:** Minor — reliability (quorum grading + health gate)
**Previous:** v2.18.0

## Highlights

- **A consultation that quietly ran with fewer panelists no longer looks authoritative.** It's now graded **MET / DEGRADED / FAILED**, and the report lists exactly which consultants dropped and why.
- **Opt-in health gate** prunes non-responsive consultants *before* the run, so you don't spend the full consultation on CLIs that are installed but not authenticated.

These were chosen from a multi-agent investigation of the "flaky CLI" problem; the adversarial review explicitly steered us *away* from an automatic CLI→API fallback (it would emit persona-less, confidence-5 stubs that poison the vote).

## What's New

### Quorum grading + Diagnosed Failures

After the panel responds, the run is graded by how many consultants actually answered, relative to `QUORUM_MIN` (default 2):

- **MET** — everyone responded
- **DEGRADED** — some failed, but ≥ quorum
- **FAILED** — below quorum

The report now opens with an `Outcome:` line and, when anything dropped, a **Diagnosed Failures** table:

```
**Outcome**: DEGRADED (5/7 responded, quorum=2)

## Diagnosed Failures
| Consultant | Reason |
|------------|--------|
| Cursor | Cursor CLI not found (command: agent) |
| Kimi   | no output and no error captured — CLI likely missing or not authenticated |
```

`QUORUM_ACTION=stop` aborts a below-quorum run; the default `warn` continues with the banner so nothing silently passes as a full-panel result.

### Health gate (opt-in)

```bash
ENABLE_HEALTH_GATE=true ai-consultants "your question"
```

Pings every selected consultant **in parallel** with a tiny real query (`HEALTH_GATE_TIMEOUT`, default 30s) and drops the ones that don't respond before the real run starts — catching installed-but-unauthenticated CLIs and stale installs. It prunes; it does not switch transport. Custom API agents without a query script are kept (not probeable).

## Breaking Changes

None. Both features are additive; `QUORUM_ACTION` defaults to `warn` (banner only) and the health gate is off by default.

## Not included (deliberately)

The investigation rejected, for now: automatic CLI→API fallback (vote-poisoning persona loss), cold-start retry-timeout escalation (stretches the whole round's worst case), warm-up calls (extra billing/OAuth contention), and a transport-abstraction circuit breaker (racy on-disk state).
