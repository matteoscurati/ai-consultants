# Release v2.21.1

**Date:** 2026-07-16  
**Type:** Patch — Kimi K3 model selection and consultant-roster cleanup  
**Previous:** v2.21.0

## Highlights

- Kimi consultations now use **K3** through the official `kimi-code/k3` CLI alias.
- AI Consultants passes the model explicitly to Kimi instead of relying on the
  user's `config.toml` default.
- Every model tier, configuration example, cost fallback, and public-facing
  surface now points to K3.
- The supported panel is reduced from 15 to **11 consultants**: Kilo, Aider,
  Amp, and Ollama are no longer shipped or selectable.

## Removed consultants

| Removed consultant | Removed command/configuration | Migration |
|---|---|---|
| Kilo | `kilocode`, `ENABLE_KILO`, `KILO_*` | Use Cursor, Qwen3, or another remaining CLI consultant. |
| Aider | `aider`, `ENABLE_AIDER`, `AIDER_*` | Use Codex or Qwen3 for implementation-focused review. |
| Amp | `amp`, `ENABLE_AMP`, `AMP_*` | Use Claude, Gemini, or Cursor according to the task. |
| Ollama | `ollama`, `ENABLE_OLLAMA`, `OLLAMA_*`, `local` preset | Local-model transport is no longer supported by this release. |

The cleanup covers the complete execution surface: canonical roster, defaults,
presets, smart routing, persona lookup, debate and synthesis, reflection,
doctor/preflight diagnostics, the CLI updater, configuration wizard, schemas,
tests, and current documentation. The four deleted `query_*.sh` adapters are not
included in the npm package.

## What changed

The runtime default is now:

```dotenv
ENABLE_KIMI=true
KIMI_CMD=kimi
KIMI_MODEL=kimi-code/k3
```

The adapter invokes Kimi non-interactively with an explicit model:

```bash
kimi --model kimi-code/k3 \
  --prompt "..." \
  --output-format stream-json
```

This matters because Kimi normally falls back to the model stored in the
user-level configuration file. Before v2.21.1, `KIMI_MODEL` was written into
response metadata but was not included in the CLI invocation, so a stale local
default could still select the previous model.

## Upgrade

If your configuration mentions a removed consultant, delete its `ENABLE_*`,
`*_MODEL`, `*_CMD`, timeout, API, and endpoint variables. Unknown legacy
variables are ignored, but removing them prevents misleading configuration.

Then pull or install v2.21.1 and verify the CLI:

```bash
ai-consultants version
kimi provider list
ai-consultants doctor --live
```

To override K3 deliberately for one run:

```bash
KIMI_MODEL=<configured-kimi-model-alias> \
  ai-consultants "Review this architecture"
```
