# Release v2.14.2

**Date:** 2026-05-29
**Type:** Patch — Claude premium upgrade to Opus 4.8 + cost-catalog unit fix
**Previous:** v2.14.1

## Highlights

- Claude premium tier upgraded from `claude-opus-4-7` to `claude-opus-4-8` (Anthropic's newest flagship).
- Pricing is unchanged — Opus 4.8 stays at $5/$25 per MTok, identical to every Opus version since 4.5.
- **Fixed a ~1000× cost-reporting overstatement** for premium/standard models: `cost_rates.json` mixed per-MTok and per-1K values, but the engine expects per-1K. The whole catalog is now normalized to per-1K.
- Swept up stale `opus-4.6`/`sonnet-4.6` short-alias defaults left over from the v2.10.6 canonical-ID migration.

## What's New

### Claude Opus 4.8 Upgrade

Anthropic released Claude Opus 4.8 as their newest flagship model. AI Consultants now selects it for the Claude premium tier — used by the default `CLAUDE_MODEL`, the `premium`/`max`/`best` tier in `get_model_for_tier`, and the `max_quality` preset.

| Tier | Before (v2.14.1) | After (v2.14.2) |
|------|-------------------|------------------|
| Premium | claude-opus-4-7 ($5/$25 per M) | claude-opus-4-8 ($5/$25 per M) |
| Standard | claude-sonnet-4-6 ($3/$15 per M) | claude-sonnet-4-6 (unchanged) |
| Economy | claude-haiku-4-5 | claude-haiku-4-5 (unchanged) |

The standard and economy tiers are untouched — Sonnet 4.6 and Haiku 4.5 remain the latest models in their respective classes. The previous premium model `claude-opus-4-7` was moved to the legacy section of `docs/cost_rates.json` so cost calculation continues to work for cached/historical responses and pinned `CLAUDE_MODEL=claude-opus-4-7` overrides.

### Cost-Catalog Unit Fix (~1000× overstatement)

`lib/costs.sh` computes cost as `(token_count / 1000) * rate`, so every value in `cost_rates.json` must be **USD per-1K tokens** (the hardcoded fallback table in `costs.sh`, e.g. `claude-3-haiku → 0.00025`, confirms this). But the premium and standard blocks had been populated with **per-MTok** dollar figures (`claude-opus-4-7: 5.00`, `claude-sonnet-4-6: 3.00`, `gpt-5.5: 3.00`, …). Result: the tool reported roughly **1000× the real cost** for almost every premium/standard model — a ~1k-in/1k-out Opus query showed up as **$30** instead of **$0.03**.

The economy block (including `claude-haiku-4-5: 0.001`) was already correct per-1K. This release normalizes the **entire** catalog — premium, standard, economy, legacy, and `default_rate` — to per-1K, and adds a `_comment_units` field documenting the contract so it can't silently regress.

Verified after the fix: Opus 1k+1k = $0.03 and 1M+1M = $30; Haiku 1k+1k = $0.006; gpt-5.5 1M+1M = $15; gemini-pro 1M+1M = $6.25 — all matching provider pricing. `docs/COST_RATES.md` Claude rows were synced to the corrected per-1K values; a few non-Claude rows in that human-facing doc may still lag and are flagged for a follow-up (`cost_rates.json` is the runtime source of truth).

### Stale Short-Alias Cleanup

The v2.10.6 release migrated Claude from short aliases (`opus-4.6`) to canonical IDs (`claude-opus-4-7`), but several spots were missed:

- `scripts/query_claude.sh` — header comment and the `MODEL_USED` fallback both still said `opus-4.6`
- `.env.example` — the `CLAUDE_MODEL=opus-4.6` template value
- `references/configuration.md` — the `CLAUDE_MODEL` override example
- `README.md` — the "Models by Tier" table Claude row
- `lib/api.sh` — `build_anthropic_request` defaulted its `model` arg to the short alias `sonnet-4.6`

The Opus spots now use `claude-opus-4-8` (and `claude-sonnet-4-6` / `claude-haiku-4-5` for the README row); the `api.sh` default is now `claude-sonnet-4-6`. All were latent in practice — callers always pass an explicit model and `config.sh` always exports `CLAUDE_MODEL` before the query script runs — but they would have surfaced if a script were invoked standalone or the example `.env` were copied verbatim.

## Breaking Changes

None. Existing `CLAUDE_MODEL` overrides continue to work — only the default changed. `claude-opus-4-7` still resolves for cost lookups via the legacy catalog block.

## Known Issues (documented, not introduced)

- The unit fix corrects the *systematic* 1000× error, but it does not re-derive every model's *individual* price against authoritative provider data. A few legacy/non-Claude per-model rates (e.g. some Mistral/DeepSeek/GLM tier orderings) reflect pre-existing catalog figures that couldn't be verified here and may not be exact. Claude rates are authoritative ($5/$25, $3/$15, $1/$5 per MTok for Opus 4.8 / Sonnet 4.6 / Haiku 4.5).
- A few non-Claude rows in `docs/COST_RATES.md` (human-facing doc) may still lag `cost_rates.json`. The JSON is the runtime source of truth.

Both are flagged for a future dedicated pricing-accuracy pass.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull

# If installed via curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

No configuration changes required. **However**, because cost figures for premium/standard models drop ~1000× with this fix, anyone who tuned `MAX_SESSION_COST` or `ENABLE_COST_AWARE_ROUTING` thresholds against the old inflated numbers should revisit those values.
