# Release v2.19.2

**Date:** 2026-07-11
**Type:** Patch — cost-tracking resilience (fresh-install fix + concurrency + corruption self-heal)
**Previous:** v2.19.1

## Highlights

- **Fresh-install cost tracking is fixed** — sessions no longer fail to record their cost the first time you run the tool.
- **A corrupt `costs.json` now self-heals** instead of permanently wedging every future cost update.
- **Concurrent consultations can't lose cost records** — the update is serialized and uses a unique temp file.
- **Cost bookkeeping can never abort a run** — every failure path degrades to a warning.
- No configuration changes; no impact on existing installs whose data dir already exists.

## What's New

### Cost tracking resilience (`lib/costs.sh`)

`track_session_cost` records each consultation's cost to `costs.json` under the XDG data dir (`~/.local/share/ai-consultants/`). Four problems are fixed:

| Problem | Before | After |
|---|---|---|
| **Fresh install** | Wrote to `costs.json` without creating its parent dir → every session failed with "No such file or directory"; cost tracking silently never accumulated | `mkdir -p` the parent dir on demand |
| **Corrupt file** | A malformed `costs.json` failed every future `jq` update, forever | Detected via `jq empty`, moved aside to `costs.json.corrupt`, and reset |
| **Concurrency** | Unguarded read-modify-write; parallel runs could interleave and lose records | Serialized with a portable `mkdir` lock (5s bounded wait, then proceeds unlocked) |
| **Temp file** | Fixed `.tmp` sibling shared across runs → clobbered writes / failed `mv` | Unique `mktemp` temp file per write |

All failure paths now log a warning and return success: cost bookkeeping is best-effort and can never abort a consultation. This matters because the caller in `consult_all.sh` runs under `set -e` *after* every consultant has already been queried and billed — a bookkeeping hiccup must not throw away the run's results.

The function is split into an outer `track_session_cost` (directory creation + lock management) and an inner `_track_session_cost_update` (the read-modify-write, run with the lock held).

### Tests

New `test_cost_tracking_resilience` in `scripts/test_suite.sh` (5 assertions, wired into `main()`) covers the fresh-install path (costs.json created under a missing nested parent dir), cross-session accumulation (0.25 → 0.75), and the corrupt-file self-heal including the `.corrupt` backup. All 8 suites pass; shellcheck clean.
