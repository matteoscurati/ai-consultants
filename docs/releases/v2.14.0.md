## What's New

### Context optimization actually runs

`scripts/build_context.sh` has carried a full AST/chunking/symbol-compression pipeline since v2.2, but the primary call path â€” the `/ai-consultants:consult` slash command â€” instructed the invoking agent (Claude/Codex/Gemini) to **inline file contents** into the query string. The result: `build_context.sh` saw a single big question and zero `FILES`, so the pipeline never engaged. A 3000-line file passed through the system raw.

The fix is a contract change at the slash-command boundary: agents now pass file paths as **positional arguments** to `consult_all.sh` rather than inlining the contents. `build_context.sh` does the file reading itself and runs the optimizer.

**Languages with dedicated AST extractors:** Python, JavaScript, TypeScript, Bash, Go. Other languages declared in `_supports_ast_extraction` fall back to a `grep`-based generic extractor; this gap was previously invisible because the pipeline never ran and is now tracked for a future tree-sitter-backed upgrade.

### Category-aware project tree

`build_context.sh` reads the exported `QUESTION_CATEGORY` env var (set by `classify_question.sh` early in `consult_all.sh`) and skips the project-tree section for question types where the listing is noise rather than signal:

| Category | Project tree included? |
|---|---|
| `ARCHITECTURE`, `CODE_REVIEW`, `API_DESIGN`, `GENERAL` | Yes |
| `SECURITY`, `QUICK_SYNTAX`, `ALGORITHM`, `BUG_DEBUG`, `DATABASE`, `TESTING` | No |
| Unknown category | Yes (conservative) |

Override: `FORCE_PROJECT_TREE=true` brings the tree back regardless of category â€” useful when you know the structure matters for a question the classifier put in a focused bucket.

Savings on a typical repo: ~1 KB / ~250 tokens per consultation in the "tree dropped" categories, which is roughly 60% of traffic based on the affinity matrix.

### File relevance tags (`@PRIMARY` / `@CONTEXT`)

When attaching multiple files, the agent can now signal which is the focus of the question and which is ambient reference:

```bash
./scripts/consult_all.sh "Why does auth fail under load?" \
    src/auth.ts@PRIMARY src/cache.ts@CONTEXT src/logger.ts@CONTEXT
```

The rendered context file marks each entry:

```
### File: `src/auth.ts` (PRIMARY)
### File: `src/cache.ts` (CONTEXT)
```

Default if omitted: `PRIMARY`. Unknown tags fall back to `PRIMARY` with a warning. Consultants are told the difference at the top of the `## Relevant Files` section.

### `--query-file <path>`

For questions that exceed shell `ARG_MAX` (~256 KB on macOS) or contain mixed-quote payloads (JSON snippets, Python dicts), agents can write the question to a tmpfile and pass `--query-file`:

```bash
echo "Long multi-paragraph question..." > /tmp/q.txt
./scripts/consult_all.sh --query-file /tmp/q.txt src/big.py
```

`--query-file` conflicts with a positional question argument (parser-level error).

### Slash commands: file detection via agent judgment

The old slash-command instructions used a hardcoded extension list + `/`-in-token heuristic to detect file paths in the user's question. This missed `Makefile`, `Dockerfile`, dotfiles like `.env.example`, and false-positively matched URLs and regex patterns. The new instruction reads:

> Identify which strings in $ARGUMENTS refer to files the user wants the consultants to look at. Use your judgment â€” verify existence via `Glob` or `Bash ls` when uncertain.

The invoking agent is the one with full conversation context, so it's better placed than a regex to make this call.

### Test coverage

`scripts/test_context_optimization.sh` (17 assertions) is the first test suite to exercise `lib/code_optimizer.sh` and `lib/chunking.sh`. It is picked up automatically by `scripts/test_all.sh`'s `find`-based discovery. Total assertions across the now-7 suites: ~510.

## Breaking Changes

**None for end users.** Backwards-compatibility is preserved:

- The legacy slash-command behavior of inlining file contents into the query string still works â€” `build_context.sh` degrades to the no-FILES branch as before. Users who upgrade their installation but pin an old slash-command alias will see no breakage, just no optimization benefit until they pull the new slash command files.
- Direct bash users invoking `./scripts/consult_all.sh "question" file1 file2` keep getting the same behavior plus the new optimization. The `@TAG` syntax is purely additive.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull

# If installed via curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash

# If installed via npx (auto-fetches latest)
npx ai-consultants "your question"
```

No configuration changes required. The new env vars (`QUESTION_CATEGORY`, `FORCE_PROJECT_TREE`) have safe defaults; `--query-file` is opt-in.

## Known limitations

- AST extractors are dedicated for Python/JS/TS/Bash/Go only; other declared languages fall back to a `grep`-based generic extractor that produces a coarse skeleton. Tracked for a future tree-sitter integration.
- Token estimation in the context manifest uses `bytes/4`, which overestimates for whitespace-heavy code and underestimates for CJK. Tracked for replacement with tiktoken.
