# Release v2.13.0

**Date:** 2026-05-04
**Type:** Minor ‚Äî Sprawl reduction: `--suggest-preset`, XDG paths, debate optimization on by default
**Previous:** v2.12.0

## Highlights

- New `ai-consultants doctor --suggest-preset --question "..."` recommends a preset + strategy combo based on the question's category and the consultants you have installed (gated by `ENABLE_*` flags and self-exclusion). `--json` for tooling/automation.
- `ENABLE_DEBATE_OPTIMIZATION` is now `true` by default ‚Äî debate is auto-skipped when consensus is high. Reduces token spend on convergent questions; SECURITY and ARCHITECTURE still mandate full debate. Decision is based on operator experience over 4 stable releases; an empirical benchmark is tracked for v2.14.
- XDG Base Directory compliance for cache, state, and data paths. No more orphaned `/tmp/ai_consultations/` after a reboot, no more world-readable LLM transcripts on multi-tenant boxes.
- README slimmed; the 150-var env table moved to `references/configuration.md`. Most users only need `ai-consultants init` + maybe one or two overrides.

## What's New

### `doctor --suggest-preset`

```
$ ai-consultants doctor --suggest-preset --question "How can I prevent SQL injection?"
Recommended:
  ai-consultants --preset balanced --strategy security_first "How can I prevent SQL injection?"

Reason:
  SECURITY detected; 10 consultants available; debate is mandatory for SECURITY
```

Routing:

| Category | Default recommendation |
|----------|------------------------|
| SECURITY | `balanced` (or `minimal` if <5 consultants) + `security_first` |
| ARCHITECTURE | `high-stakes` (‚â•5 consultants) or `balanced` (<5) + `risk_averse` |
| QUICK_SYNTAX | `fast` + `majority` |
| ALGORITHM | `balanced` + `majority` |
| BUG_DEBUG / CODE_REVIEW / TESTING | `thorough` (‚â•5) or `balanced` (<5) + `majority` |
| DATABASE / API_DESIGN | `balanced` + `majority` |
| GENERAL (or no question) | `balanced` + `majority` (or `minimal` if <5 consultants) |

The classification reuses `classify_question.sh` (pattern-based, no API call), so it's instant and offline.

### `ENABLE_DEBATE_OPTIMIZATION=true` by default

Debate is the most expensive phase of a consultation (each consultant rebuts the others' summaries). Pre-v2.13 the optimization was opt-in because it was new and conservatively gated. After 4 minor releases of stability, promoting it to default trades a small recall hit for a 30-50% token saving on consensus answers.

Safety net unchanged: SECURITY and ARCHITECTURE always run full debate. Threshold is `DEBATE_CONFIDENCE_SPREAD_THRESHOLD=2` ‚Äî debate skipped only when confidence variance is below 2 across consultants. Set `ENABLE_DEBATE_OPTIMIZATION=false` to restore pre-v2.13 behavior.

### XDG Base Directory Compliance

Pre-v2.13 paths violated the freedesktop.org spec by writing transient and persistent state to `/tmp/`:

| Var | Pre-v2.13 | v2.13 default | XDG kind |
|-----|-----------|---------------|----------|
| `DEFAULT_OUTPUT_DIR_BASE` | `/tmp/ai_consultations` | `~/.cache/ai-consultants/consultations` | cache |
| `CACHE_DIR` | `/tmp/ai_consultants_cache` | `~/.cache/ai-consultants/cache` | cache |
| `RATE_LIMIT_DIR` | `/tmp/ai_consultants_ratelimit` | `~/.cache/ai-consultants/ratelimit` | cache |
| `CHUNK_TEMP_DIR` | `/tmp/ai_consultants_chunks` | `~/.cache/ai-consultants/chunks` | cache |
| `SESSION_DIR` | `/tmp/ai_consultants_sessions` | `~/.local/state/ai-consultants/sessions` | state |
| `COST_TRACKING_FILE` | `/tmp/ai_consultants_costs.json` | `~/.local/share/ai-consultants/costs.json` | data |

Drivers:
- **Reboot loss**: `/tmp` is wiped on most distros. Cumulative cost tracking and session continuity were silently lost.
- **Multi-tenant security**: `/tmp` is world-readable on shared boxes; LLM transcripts can include pasted code and secrets.
- **`noexec` and size-capped tmpfs**: failed silently when caches grew.

All `XDG_*_HOME` env vars are honored. Falls back to `~/.{cache,local/state,local/share}/ai-consultants/` per spec, then to `/tmp/ai-consultants-{kind}` for distroless containers without HOME.

To restore pre-v2.13 behavior selectively:
```bash
export DEFAULT_OUTPUT_DIR_BASE=/tmp/ai_consultations
export SESSION_DIR=/tmp/ai_consultants_sessions
# etc.
```

Or mass-override via the user config dir (`~/.config/ai-consultants/.env`).

### `get_xdg_dir()` Helper

`lib/user_config.sh` now exports `get_xdg_dir cache|state|data` ‚Äî single source of truth for XDG resolution, used by `config.sh` to build the new defaults. No more inline `${XDG_CACHE_HOME:-$HOME/.cache}` constructions scattered across scripts.

### README Slim

The env-var section in `README.md` is now a 5-knob quick reference (`DEFAULT_PRESET`, `DEFAULT_STRATEGY`, `ENABLE_DEBATE`, `ENABLE_OLLAMA`, `MAX_SESSION_COST`) plus a pointer to `references/configuration.md` for the full ~150-var list. The first instruction is `ai-consultants init` instead of "set these env vars".

### Tests

`scripts/test_doctor.sh` (new) ‚Äî 13 assertions in 7 tests:
- `--suggest-preset` with no question defaults to GENERAL
- SECURITY ‚Üí `security_first` strategy
- QUICK_SYNTAX ‚Üí `fast` preset
- ARCHITECTURE ‚Üí `risk_averse` strategy
- ALGORITHM ‚Üí `balanced` + `majority`
- Long questions are truncated in display
- Short-circuits before running the main 22-check pipeline

## Pre-Release Review Findings (and fixes)

Four reviewer agents audited Phase 4 before tagging. All HIGH and MED findings were addressed:

| Severity | Issue | Fix |
|----------|-------|-----|
| HIGH | `suggest_preset()` swallowed classifier failures (`2>/dev/null \| tail -1`) ‚Äî degraded to GENERAL silently. | Captures stderr to tmpfile, surfaces `Warning: classification failed` with the underlying error. Drops `tail -1` (classifier prints exactly one line; trust the contract or fail loudly). |
| HIGH | XDG path defaults in `config.sh` silently regressed to `/tmp` if `get_xdg_dir` was missing (`$(... 2>/dev/null \|\| echo /tmp/ai_consultants)`). The headline v2.13 feature would have broken silently on corrupt installs. | `config.sh` now hard-fails with `FATAL` if `get_xdg_dir` is undefined. Removed the `\|\| echo /tmp/...` fallback from all 6 path defaults. |
| HIGH | `_count_available_consultants` ignored `ENABLE_*` flags and self-exclusion ‚Äî counted phantom consultants, fired wrong `_recommend_combo` boundaries. | Counter now gates each consultant on `ENABLE_<NAME>` (with sensible defaults: on for the original 6, off for opt-ins) and subtracts the invoker via `get_self_consultant_name`. |
| MED | `_recommend_combo` produced unrunnable suggestions for count<2 (e.g. `minimal` preset with 0 consultants). | Short-circuit at top: when count<2, always return the "install more CLIs" hint regardless of category. |
| MED | SECURITY count<5 fallback hardcoded "Mistral (Devil's Advocate) prioritized" without checking install. | Mistral note only emitted if `vibe` is installed AND `ENABLE_MISTRAL=true`. |
| MED | `--suggest-preset` was human-only output; tooling parsing the text would lock in the format. | Added `--json` (combine with `--suggest-preset --json`). Schema: preset, strategy, category, consultants_available, reason, classification_failed, classifier_error, question. |
| MED | Zero tests for `get_xdg_dir()` despite being the single source of truth for 6 path defaults. | 4 new tests in `test_user_config.sh` covering env override, ~/.{cache,local/state,local/share} fallback, distroless /tmp fallback, invalid kind. |
| MED | Zero tests for XDG path defaults in `config.sh`. | 2 new tests asserting all 6 paths resolve via XDG_*_HOME and that explicit env vars override. |
| MED | Zero tests for `ENABLE_DEBATE_OPTIMIZATION=true` default flip. | 1 new test asserting clean-env default. |
| LOW | Architect flagged `_recommend_combo` as a hardcoded matrix ‚Äî same anti-pattern v2.11.0 condemned. | Tracked for v2.14 as `references/preset_advisor.json` extraction. Not blocking ‚Äî consistency call, not correctness. |
| LOW | Changelog claimed "30-50% token saving" without an in-tree benchmark. | Softened to "reduces token spend ... based on operator experience"; benchmark task tracked for v2.14. |

Known gaps deferred to v2.14:
- BUG_DEBUG, CODE_REVIEW, TESTING, DATABASE, API_DESIGN categories have implicit (parametrized-by-environment) coverage in `test_doctor.sh` but no explicit boundary tests.
- `_recommend_combo` extraction to `references/preset_advisor.json` (consistency with v2.11.0 affinity matrix decision; deferred during round-1 review as LOW, escalated to MED by round-2 because each release adds branches).
- `assert_eq`/`assert_match` are duplicated across 3 test files. Extract to `lib/test_helpers.sh` to prevent drift.
- Empirical benchmark for `ENABLE_DEBATE_OPTIMIZATION=true` (changelog claim is "operator experience" only).

## Round-2 Review Findings (and fixes)

The post-fix audit caught one regression that the round-1 fix introduced and several smaller issues:

| Severity | Issue | Fix |
|----------|-------|-----|
| HIGH | Round-1 self-exclusion fix in `_count_available_consultants` was dead code: `get_self_consultant_name()` returns UPPERCASE (`CLAUDE`, `GEMINI`) but counter entries were mixed-case (`Claude`, `Gemini`). The `==` comparison never matched. | Compare against `to_upper(name)` on both CLI and API loops. New regression test (Test 13) asserts `INVOKING_AGENT=claude` drops the count. |
| MED | `--suggest-preset --json` short-circuited before the main `check_dependencies` jq probe; missing jq aborted with cryptic `command not found`. | Pre-flight jq with a clear `ERROR: --json requires 'jq' to be installed` and exit 1. New test (Test 14). |
| MED | `_count_available_consultants` had a misleading "default-true" hardcoded list that contradicted `config.sh` (e.g. claimed Aider was default-on). Drift trap, not yet a bug. | Removed the local default block; rely on `${!flag:-false}` and `config.sh` as single source of truth. |
| MED | New tests in `test_doctor.sh` didn't trap-cleanup their `mktemp -d` ‚Äî leaked tmp dirs if assertions ever started aborting. | Cleanup made explicit (and robust against bash 3.2's RETURN-trap-with-local quirk). |
| MED | 5 standalone `test_*.sh` scripts not wired into any runner; release would ship tests that never gate anything. | New `scripts/test_all.sh` aggregates all suites; `npm test` script in `package.json`. |
| LOW | `--json` schema lacked `schema_version` and `recommended_command` ‚Äî would force consumers to reconstruct the invocation string client-side and have no signal for future schema evolution. | Added both fields. New test (Test 15) asserts shape. |

## Breaking Changes

**Path defaults moved out of `/tmp`**. If your tooling parsed cost data from `/tmp/ai_consultants_costs.json` or watched `/tmp/ai_consultations/` for output, update the paths or pin the env vars (see "Restore pre-v2.13" above). For consultations themselves, the new path persists across reboots ‚Äî that's a feature, not a regression.

**`ENABLE_DEBATE_OPTIMIZATION` default flip**. Previously `false` (opt-in), now `true`. If you relied on always-debate behavior outside SECURITY/ARCHITECTURE, set `ENABLE_DEBATE_OPTIMIZATION=false`.

## Upgrade Guide

```bash
# git clone
cd ~/.claude/skills/ai-consultants && git pull

# npx
npx ai-consultants@latest version    # 2.13.0

# curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

Verify:

```bash
ai-consultants version                        # 2.13.0
ai-consultants doctor                         # 22/22 (or 21/21 without user config dir)
ai-consultants doctor --suggest-preset --question "should I use Redis or Memcached?"
./scripts/test_doctor.sh                      # 13 assertions
```

If you want to migrate persistent data from `/tmp` (sessions, cost tracking) before they're lost to a reboot:

```bash
mkdir -p ~/.local/state/ai-consultants
[ -d /tmp/ai_consultants_sessions ] && \
  cp -r /tmp/ai_consultants_sessions ~/.local/state/ai-consultants/sessions
mkdir -p ~/.local/share/ai-consultants
[ -f /tmp/ai_consultants_costs.json ] && \
  cp /tmp/ai_consultants_costs.json ~/.local/share/ai-consultants/costs.json
```

No configuration changes required for the common case.
