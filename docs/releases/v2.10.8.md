# Release v2.10.8

**Date:** 2026-05-03
**Type:** Patch — Cost catalog sync (fixes v2.10.6 drift)
**Previous:** v2.10.7

## Highlights

- Fixes a runtime drift introduced by v2.10.6: `docs/cost_rates.json` was still pointing `consultant_fallbacks` at the old model IDs (`opus-4.6`, `gpt-5.3-codex`, `composer-1.5`, `deepseek-reasoner`)
- Adds price entries for the v2.10.6 model IDs (Claude canonical IDs, `gpt-5.5`, `composer-2`, `deepseek-v4-pro`, `nvidia/nemotron-3-super-120b-a12b:free`)
- Old IDs preserved in the legacy section so cost calculation still works for historical responses or pinned overrides

## What's New

### Cost Catalog Sync

In v2.10.6 we updated `scripts/config.sh` to use canonical model IDs for Claude and to switch Codex/Cursor/Aider/DeepSeek to newer models. However, `docs/cost_rates.json` — which is read at runtime by `lib/costs.sh` to resolve fallback prices when a consultant doesn't report its own model — was not updated. Two effects:

1. `consultant_fallbacks.claude = "opus-4.6"` would have caused fallback cost lookups to use `opus-4.6` even when the consultant ran on `claude-opus-4-7`. The price for `claude-opus-4-7` was missing from the catalog entirely, which would have triggered the `default_rate` ($0.005/$0.015 per 1k) for any direct lookup of the new ID.
2. The reference `model_tiers` table in the JSON file (documentation, not used at runtime) was inconsistent with the canonical source in `config.sh::get_model_for_tier()`.

Both issues are now fixed. `consultant_fallbacks` and `model_tiers` mirror the active defaults in `config.sh`, and every model ID referenced by the defaults has a corresponding price entry.

### Catalog Layout Changes

| Section | Added | Moved to legacy |
|---------|-------|------------------|
| Premium | `claude-opus-4-7`, `gpt-5.5`, `composer-2`, `deepseek-v4-pro`, `nvidia/nemotron-3-super-120b-a12b:free` | `opus-4.6`, `gpt-5.3-codex`, `composer-1.5`, `deepseek-reasoner` |
| Standard | `claude-sonnet-4-6` | `sonnet-4.6` |
| Economy | `claude-haiku-4-5` | `haiku-4.5` |

Pricing values mirror the prior entries for direct successors (the model lineage is the same, only the canonical ID changed). `nvidia/nemotron-3-super-120b-a12b:free` is set to $0/$0 to reflect OpenRouter's free tier — this means Aider consultations contribute zero to cost tracking when running on the default model.

`COST_RATES.md` was updated to mirror the new active and legacy IDs.
