## What's New

### Antigravity CLI (`agy`) replaces the Gemini CLI

Google's 2026-06-18 transition retired the Gemini CLI for non-enterprise users. The Gemini consultant now invokes `agy`:

| | Before (Gemini CLI) | After (Antigravity CLI) |
|---|---|---|
| Binary | `gemini` | `agy` |
| Install | `npm install -g @google/gemini-cli` | `curl -fsSL https://antigravity.google/cli/install.sh \| bash` |
| Model flag | `-m gemini-3.1-pro-preview` | `--model "Gemini 3.1 Pro (High)"` |
| Output | `--output-format json` (CLI envelope) | plain text (the model's JSON directly) |
| Auth | `GEMINI_API_KEY` / `gemini auth login` | OAuth â€” run `agy` once to sign in (cached) |

The persona instruction already forces the model to emit our JSON schema, so agy's plain-text output *is* the structured envelope â€” no wrapper to unwrap. Internally we dropped the `--output-format json` flag and the `native_json_field` extraction that the old CLI envelope required.

### Model tiers (agy display names)

| Tier | Model |
|------|-------|
| premium (default) | `Gemini 3.1 Pro (High)` |
| standard | `Gemini 3.5 Flash (High)` |
| economy | `Gemini 3.5 Flash (Low)` |

Run `agy models` (after signing in) to see all available models, including Claude and GPT-OSS backends that agy also exposes.

### API mode unchanged, now with its own model var

API mode (`GEMINI_USE_API=true`) still calls the Google AI endpoint and is unaffected by the CLI deprecation. It now reads its model ID from the new **`GEMINI_API_MODEL`** (default `gemini-3.1-pro-preview`) instead of `GEMINI_MODEL`, because the endpoint needs an API ID, not an agy display name.

## Breaking Changes

- **The Gemini consultant now requires the `agy` CLI and an OAuth sign-in.** If you previously relied on the `gemini` binary or `GEMINI_API_KEY` for headless CLI use, install `agy` and run it once to authenticate. Enterprise users with continued Gemini CLI access can opt out by setting `GEMINI_CMD=gemini` and `GEMINI_MODEL=<api-id>`.
- `GEMINI_MODEL` default changed from `gemini-3.1-pro-preview` to `Gemini 3.1 Pro (High)`. If you pinned `GEMINI_MODEL` to an API ID for CLI use, update it to an agy display name (or move it to `GEMINI_API_MODEL` if you use API mode).

## Upgrade Guide

```bash
# 1. Update the skill
cd ~/.claude/skills/ai-consultants && git pull
# (or) curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash

# 2. Install the Antigravity CLI and sign in
curl -fsSL https://antigravity.google/cli/install.sh | bash
agy            # launch once to complete OAuth sign-in

# 3. Verify
ai-consultants doctor      # should show "Gemini: <agy version>"
```

No configuration changes are required if you use defaults. To pin a model: `export GEMINI_MODEL="Gemini 3.1 Pro (High)"`.
