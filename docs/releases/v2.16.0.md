# Release v2.16.0

**Date:** 2026-06-22
**Type:** Minor — dynamic orchestration engine
**Previous:** v2.15.1

## Highlights

- **The panel now deliberates adaptively.** A planner picks an orchestration *shape* for each question, and debate runs as a **convergence loop** — it iterates until the consultants' answers converge instead of for a fixed number of rounds.
- **Five shapes**, auto-selected from the question: `quick`, `converge`, `adversarial`, `tournament`, `exhaustive`.
- **Security questions get an adversarial refutation gate**; "compare X vs Y" questions run a tournament that declares one winner; "find all / audit" questions loop until no new angle surfaces.
- **Fully standalone bash** — inspired by Claude Code's dynamic workflows, but no dependency on the Workflow tool or any host. Works the same from Claude Code, Codex, Gemini, or a plain shell.
- **Backwards-compatible.** `ORCHESTRATION_MODE=fixed` restores the exact pre-2.16 pipeline. Budget limits still cap every round.

## What's New

### From a fixed pipeline to a planned one

Until now ai-consultants always ran the same sequence: classify → query the panel → a fixed number of debate rounds → synthesize. v2.16.0 inserts a **planner** (`lib/orchestration.sh`) between classification and deliberation. It reads the question's category, complexity, and intent and chooses how the panel should work:

| Shape | Auto-selected when | What it does |
|-------|--------------------|--------------|
| `quick` | low complexity | single fan-out, no debate |
| `converge` | medium/high complexity (default) | debate until consensus reaches the target |
| `adversarial` | category is `SECURITY` | forces ≥1 critique round, then runs anonymous peer review as a refutation gate |
| `tournament` | intent is "compare X vs Y" | converges, then synthesis declares a single winning approach with pairwise reasoning |
| `exhaustive` | intent is "find all / audit" | loops until a round surfaces no new approach |

### Convergence instead of fixed rounds

Debate is no longer a hardcoded count. The convergence loop runs rounds until one of:
- consensus ≥ `CONVERGENCE_TARGET_CONSENSUS` (converged),
- the per-round consensus gain drops below `CONVERGENCE_STALL_EPSILON` (stalled — the panel has stopped moving),
- `CONVERGENCE_MAX_ROUNDS` is hit, or
- the budget would be exceeded.

Each run records its consensus trajectory and stop reason to `orchestration.json`, and the chosen shape/complexity/intent to `optimization_metrics.json` — no silent caps.

### Settings

```bash
ORCHESTRATION_MODE=auto              # auto (planner) | fixed (legacy) | quick|converge|adversarial|tournament|exhaustive
CONVERGENCE_MAX_ROUNDS=4
CONVERGENCE_TARGET_CONSENSUS=75      # 0-100
CONVERGENCE_STALL_EPSILON=5
ENABLE_ADVERSARIAL_VERIFY=true
```

## Breaking Changes

None. The default changes behavior (deliberation is now adaptive rather than fixed-round), but `ORCHESTRATION_MODE=fixed` restores the previous pipeline exactly, and `MAX_SESSION_COST` / budget enforcement still bounds every round.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull

# If installed via curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

No configuration changes required. To keep the old fixed-round behavior, set `ORCHESTRATION_MODE=fixed`. To force a specific shape, set `ORCHESTRATION_MODE=converge` (or `adversarial`, `tournament`, `exhaustive`, `quick`).
