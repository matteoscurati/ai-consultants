# Release v2.18.0

**Date:** 2026-06-26
**Type:** Minor — observability (diagnosable failures + live doctor)
**Previous:** v2.17.2

## Highlights

- **When a consultant fails, you now see *why*** — "CLI not found", "401 Unauthorized", "Timeout", etc. — instead of a bare "Failed".
- **`doctor.sh --live`** sends a real ping to each enabled consultant, catching CLIs that are installed but not authenticated (which the static check reports as healthy).

## What's New

### Failures tell you the reason

Previously the orchestrator launched every consultant with its output **and stderr** sent to `/dev/null`, so a failing consultant produced only `Failed` / `Empty response` with no detail — making it impossible to distinguish *not installed* from *not authenticated* from a *transient* error without digging through temp files. (In a real session this led to a wrong diagnosis — blaming a timeout for what were actually auth failures.)

Now each consultant's stderr is captured, and on failure the run prints a one-line reason:

```
  ✗ Cursor: Failed
    ↳ Cursor: [ERROR] Cursor CLI not found (command: agent)
  ✗ GLM: Failed
    ↳ GLM: All 2 attempts failed: 401 Unauthorized
```

If nothing diagnostic was captured, it says so plainly ("no output and no error captured — CLI likely missing or not authenticated").

### `doctor.sh --live`

The normal `doctor.sh` only checks that a consultant's CLI is *installed* (`--version`). `--live` goes further: it sends a minimal real query to each **enabled** consultant and reports which actually respond.

```bash
./scripts/doctor.sh --live
# ✓ Gemini: responded
# ✗ Cursor: no valid response — Cursor CLI not found (command: agent)
# ✗ Kimi: no valid response
# Live result: 1 responding, 2 failing
```

It's opt-in (one small query per consultant) and honors `DOCTOR_LIVE_TIMEOUT` (default 45s). Failures are reported as `doctor` issues (exit 1).

## Breaking Changes

None.

## Not fixable in code

`--live` and the surfaced reasons *diagnose* a consultant that's missing or unauthenticated — but installing/authenticating that CLI is up to you. Cold-start/warm-up retry effects are also environmental.
