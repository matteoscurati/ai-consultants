# Release v2.13.1

**Date:** 2026-05-04
**Type:** Patch — Perf, DRY, latent-bug fixes, shellcheck cleanup, CI coverage
**Previous:** v2.13.0

## Highlights

- **~200-400ms saved per consultation** by exporting XDG roots once at first `config.sh` source — child `query_*.sh` subshells inherit the values and skip ~84 subshell forks (6 path lookups × 14 consultants).
- **6 test suites in `npm test`** (was 5): added `test_suite.sh` (258 library assertions). Total ~493 assertions gating CI now.
- **5 lib files realigned** with the v2.13 XDG migration — pre-fix they still defaulted to `/tmp/ai_consultants_*` and would have silently regressed if sourced standalone.
- **0 shellcheck warnings** under the project exclusions (was 37).
- Zero behavioral change for end users.

## What's New

### Perf: XDG Cache Export

v2.13.0 introduced XDG-compliant defaults but resolved them via `$(get_xdg_dir cache)` per `${VAR:-...}` evaluation. `config.sh` is sourced 15-30 times per consultation transitively (each `query_*.sh` runs in a backgrounded subshell that re-sources `lib/common.sh` → `config.sh`), and each source re-ran the 6 subshells for the 6 path defaults — ~84 forks per consultation × ~3-5ms each on macOS = **~200-400ms tax** that disappeared into the wall-clock noise.

v2.13.1 resolves the 3 XDG roots **once** and exports them:

```bash
: "${_AI_CONSULTANTS_XDG_CACHE:=$(get_xdg_dir cache)}"
: "${_AI_CONSULTANTS_XDG_STATE:=$(get_xdg_dir state)}"
: "${_AI_CONSULTANTS_XDG_DATA:=$(get_xdg_dir data)}"
export _AI_CONSULTANTS_XDG_CACHE _AI_CONSULTANTS_XDG_STATE _AI_CONSULTANTS_XDG_DATA
```

Child subshells inherit the values; subsequent sources hit the parameter-expansion no-op path and skip the subshell entirely. Verified empirically.

### Perf: Stagger and Counter Microfixes

- `apply_launch_stagger()` no longer forks `awk` — pure-bash `printf '%d.%03d'`. Saves 14 forks per consultation.
- `_count_available_consultants` entries pre-uppercased to match `get_self_consultant_name`'s output format. Eliminates 15 `to_upper` subshells per `--suggest-preset`. (Round-2 review fix carried into the perf model.)

### Latent Bug Fix: `lib/*.sh` Defaults Drift

v2.13.0 migrated path defaults from `/tmp` to XDG dirs in `config.sh`, but five lib files still carried the legacy `/tmp/ai_consultants_*` fallbacks:

| File | Was | Now |
|------|-----|-----|
| `lib/cache.sh:13` | `/tmp/ai_consultants_cache` | `${_AI_CONSULTANTS_XDG_CACHE:-/tmp/ai_consultants}/cache` |
| `lib/session.sh:12` | `/tmp/ai_consultants_sessions` | `${_AI_CONSULTANTS_XDG_STATE:-/tmp/ai_consultants}/sessions` |
| `lib/api.sh:561` | `/tmp/ai_consultants_ratelimit` | `${_AI_CONSULTANTS_XDG_CACHE:-/tmp/ai_consultants}/ratelimit` |
| `lib/chunking.sh:31` | `/tmp/ai_consultants_chunks` | `${_AI_CONSULTANTS_XDG_CACHE:-/tmp/ai_consultants}/chunks` |
| `lib/costs.sh:348` | `/tmp/ai_consultants_costs.json` | `${_AI_CONSULTANTS_XDG_DATA:-/tmp/ai_consultants}/costs.json` |

In production they were dead code (config.sh always sets the env var first), but standalone sourcing (e.g. a future targeted test) would have silently put state in `/tmp` instead of `~/.cache`/`~/.local/state`/`~/.local/share`.

`lib/session.sh::cleanup_old_sessions` also hardcoded `/tmp/ai_consultations` — now uses `$DEFAULT_OUTPUT_DIR_BASE`.

### DRY: `scripts/lib/test_helpers.sh`

The 3 standalone test files (`test_user_config.sh`, `test_doctor.sh`, `test_bin.sh`) each defined identical `assert_eq`/`assert_match`/`run_test` ~30-line blocks. Extracted to `scripts/lib/test_helpers.sh`:

- `assert_eq <expected> <actual> <message>`
- `assert_match <pattern> <haystack> <message>`
- `run_test "<name>" <function>` — calls `_reset_state` if defined, runs the test
- `test_summary "<suite_name>"` — prints summary and exits with the right code
- Counters `failed`/`checked` and color codes are file-scoped

`run_test` calls `_reset_state` automatically before each test, so suites can drop boilerplate. Future test suites just `source lib/test_helpers.sh` and define their own `_reset_state` if they manage shared state.

### CI: `test_suite.sh` Now Gated by `npm test`

`scripts/test_all.sh` originally excluded `test_suite.sh` (it was the legacy "library tester" run separately). After the v2.12-v2.13 round of reviews repeatedly flagged "tests on disk but not gating CI" as the recurring debt, this release wires it in. `test_all.sh` now runs:

|-------|-----------|

| `test_routing_parity.sh` | 146 |
| `test_set_e_safety.sh` | static lint + dynamic check |
| `test_bin.sh` | 10 |
| `test_doctor.sh` | 31 |
| `test_suite.sh` | 258 |
| **Total** | **~493** |

`npm test` runs all of them.

## Breaking Changes

None.

The new `_AI_CONSULTANTS_XDG_{CACHE,STATE,DATA}` env vars are exported but treated as internal — they are documented as "exported only for child shells of `consult_all.sh`; not part of the public API." Setting them externally is allowed and overrides the defaults.

## Upgrade Guide

```bash
# git clone
cd ~/.claude/skills/ai-consultants && git pull

# npx
npx ai-consultants@latest version    # 2.13.1

# curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

Verify:

```bash
ai-consultants version            # 2.13.1
ai-consultants doctor             # 21/21 (or 22/22 with user config dir)
npm test                          # 6 suites, ~493 assertions
```

No configuration changes required.
