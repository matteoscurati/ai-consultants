# Release v2.10.1

**Date:** 2026-03-15
**Type:** Patch — quality improvements, bug fixes, documentation
**Previous:** v2.10.0

## Highlights

- Slash commands now guide Claude through file reading, result presentation, and error recovery
- `debate_round.sh` hardened against 9 classes of silent failures
- Token usage reduced ~17% on typical invocations via progressive disclosure

## What's New

### Slash Command Quality (P0-P2)

The `/ai-consultants:consult` and `/ai-consultants:debate` commands were fully rewritten to solve three runtime problems identified by skill evaluation:

- **File context handling**: When users pass file paths (e.g., `/consult "Review this" src/auth.ts`), Claude now reads each file with the Read tool and inlines the contents into the query before calling the script. This fixes the broken `$ARGUMENTS` single-string quoting issue.
- **Result presentation**: Claude now reads `report.md` and `synthesis.json` from the output directory and presents structured results (consensus, summary table, recommendation, agreements, disagreements, risks, next steps).
- **Error recovery**: Actionable guidance for exit code 1 (→ `doctor.sh`), exit code 124 (→ `--preset fast`), partial failures, and empty output.

### debate_round.sh Hardening

Nine issues fixed, including 4 critical:

| Severity | Fix | Impact |
|----------|-----|--------|
| CRITICAL | `((count++)) \|\| true` (4 instances) | Script no longer aborts on bash 4+/5+ (Linux) |
| CRITICAL | `*` default case in consultant routing | Future consultants no longer silently ignored |
| CRITICAL | Amp, Kimi, MiniMax case entries added | All 15 consultants can now participate in debate |
| HIGH | Subshell stderr → per-consultant `.err` files | Query failures now produce diagnostic logs |
| HIGH | `$my_response` JSON validation + fallback | Corrupt responses no longer break debate prompts |
| MEDIUM | Wait loop logs which consultant failed | Users see which consultant failed, not just count |
| MEDIUM | ROUND_NUMBER integer validation | Non-integer input rejected with clear error |
| MEDIUM | Config source warning instead of silent suppression | `config.sh` load failures now visible |
| LOW | Temp file cleanup on jq failure | No more orphaned files in `/tmp` |

Content moved to `references/details.md` for on-demand loading. Presets, strategies, bash usage, and best practices are no longer loaded on every invocation.

### Documentation Audit

18 issues fixed across 11 files:

- **Consultant completeness**: Kimi added to README.md, SETUP.md, JSON_SCHEMA.md, SMART_ROUTING.md, COST_RATES.md, schema.json. MiniMax added where missing.
- **Version consistency**: Headers updated from v2.8 to v2.10 in JSON_SCHEMA.md and SMART_ROUTING.md
- **Stale defaults fixed**: OLLAMA_MODEL (`llama3.2` → `qwen2.5-coder:32b`), QWEN3_USE_API (`true` → `false`), DEBATE_CONFIDENCE_SPREAD_THRESHOLD (`2` → `3`)
- **Path corrections**: `lib/debate_round.sh` → `debate_round.sh` in CLAUDE.md
- **Consultant count**: `13` → `15` throughout SETUP.md
- **Self-exclusion**: Claude's consult.md description no longer lists itself
