# Release v2.25.1

**Date:** 2026-07-21
**Type:** Patch — six defects found by an independent review of v2.25.0
**Previous:** v2.25.0

## Highlights

- **The final score said 2/10 even when every consultant agreed.** Fixed.
- **Peer-review artifacts were being billed as consultations** — a regression introduced by v2.25.0's own cost work.
- **Smart routing queried consultants you had disabled**, and billed them.
- Gemini API-mode calls were costed as the wrong model.
- Peer review crashed on the stock macOS shell.
- Consensus reported 100% for panels that genuinely disagreed.

## What's New

### The score was wrong on essentially every consultation

The recommendation was emitted as a key with spaces and punctuation stripped, while scoring compared each consultant's original answer against that stripped version. Nothing ever matched, so every consultant was counted as a dissenter.

| Answer | Score |
|---|---|
| `Use JWT with rotating sessions` | 2/10 |
| `JWT` | 10/10 |

Same panel, same unanimity, same confidence. Real answers are multiword.

The same stripping merged genuinely different options: `Use JWT + sessions` and `Use JWT sessions` collapsed into one. Both are fixed, and the voting tests now use realistic multiword answers — using single-word ones is why this survived.

### A regression from v2.25.0

v2.25.0 made the cost scan recursive so it would reach debate rounds. That also reached everything peer review writes below the response directory: anonymized copies keep the original token counts, and bookkeeping files fall back to 1000 tokens each. A run whose real cost was $0.125 reported $0.244.

Billing now covers the responses and the debate rounds and nothing else, and a response is recognised by its structure rather than by its filename.

### Disabled consultants were being queried

`ENABLE_*` and self-exclusion were applied only on the non-smart path. With `ENABLE_SMART_ROUTING=true`, a consultant you had disabled — or the agent invoking the skill — could still be sent the question and billed. Custom API agents were excluded from smart routing entirely; they are now included.

### The rest

- **Gemini API mode** sent `GEMINI_API_MODEL` but recorded `GEMINI_MODEL`, mispricing by 2.24x.
- **Peer review** used two bash 4 constructs under `/bin/bash` 3.2, the stock macOS shell that `doctor` reports as supported.
- **Consensus** grouped answers transitively, so A resembling B and B resembling C put A and C together even with nothing in common. Groups now require their members to resemble each other, which also prevents a convergence loop ending early on false agreement.

## Breaking Changes

None. No configuration key, default, or output field changed.

Two numbers will move, both because they were wrong: the final score rises on any panel that agreed on a multiword answer, and consensus falls on panels that were being merged transitively.


No configuration changes required.

## Verification

Each fix carries a regression test observed failing before the change. `npm test` covers 22 suites.

One test in this release passed locally and failed in CI: it asserted a consultant was selected without enabling it, and `ENABLE_DEEPSEEK` defaults to false — so it held only for a maintainer whose own config enabled it. Verify with the user config neutralised, which is the condition CI runs in:

```bash
AI_CONSULTANTS_CONFIG_DIR=$(mktemp -d) npm test
```
