# Release v2.25.2

**Date:** 2026-07-22
**Type:** Patch — `doctor` stops reporting health it hasn't verified
**Previous:** v2.25.1

## Highlights

- `doctor` no longer prints "All systems healthy!" when it has only checked that CLIs are installed and keys are present.
- "healthy" is now reserved for `doctor --live`, which sends a real query.
- `--json` gains a `verified` field and a `static_ok` status so tooling can tell a real all-clear from an unverified one.

## What's New

### The static all-clear was false

The default `doctor` confirms a consultant's CLI is installed and its key is present. Neither means the consultant can actually answer — a consultant with an expired key or an exhausted quota passes every static check and fails on the first real query.

Measured on a real machine: static `doctor` reported all 11 consultants healthy, while `doctor --live` showed **8 of 11 actually respond** (the other three had a usage limit or a bad key). The static "healthy" gave a false all-clear.

Now:

- A clean static run prints, instead of "All systems healthy!":
  > `✓ Static checks passed: CLIs installed, keys present.`
  > `This does NOT confirm consultants can answer — verify with: doctor --live`
- The word **"healthy" is reserved for a `--live` run** that pinged the consultants for real.
- `--json` reports `status: "static_ok"` for a clean static run and gains a `verified: "live" | "static"` field. `status: "healthy"` now implies a live check backed it.

`--live` is unchanged, and a plain `doctor` still makes no billed calls.

## Breaking Changes

None in behavior. One output-contract change worth noting for tooling: a clean **static** `doctor --json` now returns `status: "static_ok"` (with `verified: "static"`) instead of `"healthy"`. Consumers that checked for exactly `"healthy"` on a static run should accept `static_ok`, or run `--live` to get `"healthy"`.

## Upgrade Guide

```bash
# If installed via git clone
cd ~/.claude/skills/ai-consultants && git pull

# If installed via curl | bash
curl -fsSL https://raw.githubusercontent.com/matteoscurati/ai-consultants/main/scripts/install.sh | bash
```

No configuration changes required. To verify your consultants actually respond, run `ai-consultants doctor --live`.

## Verification
